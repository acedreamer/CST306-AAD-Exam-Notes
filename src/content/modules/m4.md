## Dynamic Programming: Matrix Chain Multiplication
**[SURE SHOT | ↻ REPEATED PYQ]**

### Section A: What is this? (The Concept)
Imagine you are a chef in a high-pressure kitchen. You have three tasks: peeling potatoes, chopping them, and frying them. Depending on which tools you use and which task you do first, the total effort changes. If you chop then peel, it takes forever. If you peel first then chop, it's efficient.

> **The Analogy:** Matrix Chain Multiplication is like finding the most efficient "recipe" for multiplying a long string of matrices. Multiplying matrices is expensive (takes many scalar multiplications). Because matrix multiplication is associative, $(A \times B) \times C$ gives the same result as $A \times (B \times C)$, but one way might take 10 times more "effort" (scalar multiplications) than the other. We use Dynamic Programming to find the absolute cheapest way to parenthesize the chain.

### Section B: Exam-Ready Theory (The Rigor)

**1. Problem Statement:** Given a chain of $n$ matrices $\langle A_1, A_2, \dots, A_n \rangle$, where matrix $A_i$ has dimensions $p_{i-1} \times p_i$. The goal is to fully parenthesize the product $A_1 A_2 \dots A_n$ in a way that minimizes the total number of scalar multiplications.

**2. Cost Calculation:** Multiplying a $p \times q$ matrix by a $q \times r$ matrix takes $p \times q \times r$ scalar multiplications.

**3. Recursive Property:** Let $m[i, j]$ be the minimum cost to multiply matrices $A_i$ through $A_j$.
$$
m[i, j] = 
\begin{cases} 
0 & \text{if } i = j \\
\min_{i \leq k < j} \{ m[i, k] + m[k+1, j] + p_{i-1} p_k p_j \} & \text{if } i < j
\end{cases}
$$
Where $k$ is the split point.

**4. Tables Used:**
*   **$m[n, n]$ table:** Stores the minimum cost of subproblems.
*   **$s[n-1, n]$ table:** Stores the index $k$ that achieved the minimum cost (used for path reconstruction).

**5. Complexity:** 
*   Time Complexity: $O(n^3)$
*   Space Complexity: $O(n^2)$

### Section C: Step-by-Step Worked Example (The Application)

**Example: Matrix Chain Multiplication (April 2025 University Question)**
Chain of 4 matrices: $A_1(5 \times 6), A_2(6 \times 4), A_3(4 \times 8), A_4(8 \times 10)$.
Dimensions $p = [5, 6, 4, 8, 10]$.

1.  **$m[i,j]$ table initialization:**
    *   Diagonal $L=1$: $m[1,1]=m[2,2]=m[3,3]=m[4,4]=0$.
2.  **$L=2$ (Length 2 chains):**
    *   $m[1,2] = 5 \times 6 \times 4 = 120$.
    *   $m[2,3] = 6 \times 4 \times 8 = 192$.
    *   $m[3,4] = 4 \times 8 \times 10 = 320$.
3.  **$L=3$ (Length 3 chains):**
    *   $m[1,3] = \min($
        *   $k=1: 0 + 192 + (5 \times 6 \times 8) = 192 + 240 = 432$
        *   $k=2: 120 + 0 + (5 \times 4 \times 8) = 120 + 160 = 280$
        $)$ **Min = 280 at $k=2$**.
    *   $m[2,4] = \min($
        *   $k=2: 0 + 320 + (6 \times 4 \times 10) = 560$
        *   $k=3: 192 + 0 + (6 \times 8 \times 10) = 672$
        $)$ **Min = 560 at $k=2$**.
4.  **$L=4$ (Final Chain):**
    *   $m[1,4] = \min($
        *   $k=1: 0 + 560 + (5 \times 6 \times 10) = 860$
        *   $k=2: 120 + 320 + (5 \times 4 \times 10) = 640$
        *   $k=3: 280 + 0 + (5 \times 8 \times 10) = 680$
        $)$ **Min = 640 at $k=2$**.
5.  **Result:** Optimal scalar multiplications = 640.
6.  **Parenthesization:** $s[1,4]=2 \implies (A_1 A_2) (A_3 A_4)$.

### Section D: How to Write This in the Exam (The Strategy)
*   **Start With:** Define the problem and the cost formula $(p \times q \times r)$. State the Principle of Optimality.
*   **Body:** You **must** draw the $m$ table (triangular matrix). Fill it diagonal by diagonal. Show the split points in the $s$ table or write them as subscripts. Show at least one calculation for a cell where $L \geq 3$ in full detail.
*   **Traps:** Do not skip the "Parenthesization" part. After finding the min cost, use the $s$ table to write the final string.
*   **Close With:** Final Min Cost value and the optimal order.

## Dynamic Programming: Floyd-Warshall (All Pairs)
**[SURE SHOT | ↻ REPEATED PYQ]**

### Section A: What is this? (The Concept)
Imagine you are designing a high-speed train network for the whole country. Dijkstra's algorithm helps you find the fastest way from *one* city to everywhere else. But you need a master table that shows the fastest way between *every* possible pair of cities simultaneously.

> **The Analogy:** Floyd-Warshall is the "Master Schedule". It starts with a simple table of direct routes. Then it says: "What if we allow a stopover in City 1? Can we find a shortcut?" Then it checks City 2, then City 3, and so on. By the time it has allowed stopovers in every city, the table represents the absolute shortest path between every pair.

### Section B: Exam-Ready Theory (The Rigor)

**1. Definition:** An algorithm for finding shortest paths between all pairs of vertices in a weighted directed graph. It handles **negative edge weights** but NOT negative cycles.

**2. Core Recurrence:** Let $D^{(k)}_{ij}$ be the shortest path from $i$ to $j$ using only intermediate vertices from the set $\{1, 2, \dots, k\}$.
$$
D^{(k)}_{ij} = \min(D^{(k-1)}_{ij}, D^{(k-1)}_{ik} + D^{(k-1)}_{kj})
$$
*   **$D^{(0)}$:** Initial weight adjacency matrix (direct edges).
*   **$D^{(n)}$:** Final matrix containing all-pairs shortest paths.

**3. Complexity:**
*   Time Complexity: $O(V^3)$ (three nested loops).
*   Space Complexity: $O(V^2)$.

### Section C: Step-by-Step Worked Example (The Application)

**Example: Floyd-Warshall Trace (University Question)**
Initial Matrix $D^{(0)}$ (4 nodes):
```text
    1   2   3   4
1 [ 0,  9, -4,  α]
2 [ 6,  0,  α,  2]
3 [ α,  5,  0,  α]
4 [ α,  α,  1,  0]
```
1.  **Matrix $D^{(1)}$ (Via node 1):**
    *   $D^{(1)}_{23} = \min(D^{(0)}_{23}, D^{(0)}_{21} + D^{(0)}_{13}) = \min(\alpha, 6 + (-4)) = 2$.
2.  **Matrix $D^{(2)}$ (Via node 2):**
    *   $D^{(2)}_{13} = \min(D^{(1)}_{13}, D^{(1)}_{12} + D^{(1)}_{23}) = \min(-4, 9 + 2) = -4$.
3.  **Final Matrix $D^{(4)}$:** Contains all-pairs shortest paths.
    *   *Exam Tip: You must show each $D^{(k)}$ matrix explicitly.*

### Section D: How to Write This in the Exam (The Strategy)
*   **Start With:** Write the Floyd-Warshall algorithm (3 nested `for` loops) and the recurrence relation.
*   **Body:** Clearly label each matrix as $D^{(0)}, D^{(1)}, \dots, D^{(n)}$. For each matrix $D^{(k)}$, state: "Updating based on vertex $k$". Mention that the $k^{th}$ row and $k^{th}$ column remain unchanged during the $k^{th}$ iteration.
*   **Traps:** If an edge weight is $\infty$, be careful when adding it to another number ($\infty + x = \infty$).
*   **Close With:** Conclude that the final matrix $D^{(n)}$ provides the all-pairs shortest distances.

## Backtracking vs Branch & Bound
**[PART A FOCUS | HIGH FREQ]**

### Section A: What is this? (The Concept)
Imagine you are looking for a hidden treasure in a maze. 

> **The Analogy:** **Backtracking** is like walking into a tunnel, hitting a dead end, and turning back to try the other fork. You explore one branch completely before trying another (**DFS**). **Branch & Bound** is like sending 10 friends with walkie-talkies. They all start at the entrance and explore the closest intersections first (**BFS**). Crucially, they have "bounds"—if a friend sees a sign that says "No treasure this way," they immediately stop exploring that whole section and tell the others.

### Section B: Exam-Ready Theory (The Rigor)

| Feature | Backtracking | Branch & Bound |
| :--- | :--- | :--- |
| **Search Method** | Depth First Search (DFS) | Breadth First Search (BFS/LC-Search) |
| **State Space Tree** | Explores one branch deeply until dead end. | Explores level-by-level or based on cost. |
| **Pruning** | Uses Bounding/Criterion functions to kill branches. | Uses Bounding functions + **Cost functions** (Lower Bounds). |
| **Application** | Finding *any* solution or all feasible solutions. | Primarily used for **Optimization** (Min/Max) problems. |
| **Examples** | N-Queens, Sum of Subsets, Graph Coloring. | TSP, 0/1 Knapsack (optimization), Job Scheduling. |

## Backtracking: N-Queens Problem
**[HIGH PROB | ↻ REPEATED PYQ]**

### Section A: What is this? (The Concept)
Imagine you are trying to seat 4 angry queens on a $4 \times 4$ chessboard. These queens hate each other—if any queen can see another (horizontally, vertically, or diagonally), they will fight.

> **The Analogy:** N-Queens is the ultimate "Peacekeeper" puzzle. You place the first queen. Then you try placing the second. If they see each other, you move the second queen. If you run out of spots for the second queen, you realize the *first* queen was in a bad spot and move her. This "try, fail, move back, try again" is the essence of Backtracking.

### Section B: Exam-Ready Theory (The Rigor)
*   **Constraint 1 (Explicit):** Exactly one queen per row. We represent the solution as an $n$-tuple $(x_1, x_2, \dots, x_n)$, where $x_i$ is the column for the queen in row $i$.
*   **Constraint 2 (Implicit):**
    *   No two queens in the same column: $x_i \neq x_j$.
    *   No two queens on the same diagonal: $|i - j| \neq |x_i - x_j|$.
*   **Algorithm:** 
    1.  Place queen $k$ in row $k$ and column $i$.
    2.  Check if `Place(k, i)` is valid (no attacks).
    3.  If valid and $k=n$, solution found.
    4.  If valid and $k<n$, recursively call for $k+1$.
    5.  If not valid, try column $i+1$.

### Section C: Step-by-Step Worked Example (The Application)

**Example: Solve 4-Queens Problem**
1.  **Queen 1:** Place at (1, 1).
2.  **Queen 2:** Try (2, 1) [Fail], Try (2, 2) [Fail], Try (2, 3) [Success].
3.  **Queen 3:** Try (3, 1) [Fail], Try (3, 2) [Fail], Try (3, 3) [Fail], Try (3, 4) [Fail]. 
    *   *Backtrack to Queen 2.*
4.  **Queen 2:** Move to (2, 4) [Success].
5.  **Queen 3:** Try (3, 1) [Fail], Try (3, 2) [Success].
6.  **Queen 4:** Try (4, 1) [Fail], Try (4, 2) [Fail], Try (4, 3) [Fail], Try (4, 4) [Fail].
    *   *Backtrack to Queen 3, then Queen 2, then Queen 1.*
7.  **Queen 1:** Move to (1, 2).
8.  **Queen 2:** Try (2, 4) [Success].
9.  **Queen 3:** Try (3, 1) [Success].
10. **Queen 4:** Try (4, 3) [Success].

**Final Solution Vector:** $X = (2, 4, 1, 3)$.

### Section D: How to Write This in the Exam (The Strategy)
*   **Start With:** Explain the Explicit and Implicit constraints.
*   **Body:** You **must** draw the **State Space Tree**. Show nodes being killed (pruned) with a cross symbol. Number the nodes in the order they are generated (DFS order).
*   **Close With:** Write out the final column positions, e.g., for 4-Queens: $(2, 4, 1, 3)$.

## Branch and Bound: Traveling Salesperson Problem (TSP)
**[HIGH PROB | ↻ REPEATED PYQ]**

### Section A: What is this? (The Concept)
Imagine a salesperson who needs to visit 5 cities and return home. He wants to spend as little money as possible on tickets. 

> **The Analogy:** TSP using Branch & Bound is like "Eliminating expensive vacations". We start with a lower bound on the total cost. As we pick a route (a "branch"), we calculate a new "bound" for that route. If the cheapest possible way to finish that route is *already* more expensive than a route we already found, we abandon it immediately (killing the node).

### Section B: Exam-Ready Theory (The Rigor)
*   **Method:** Uses **Matrix Reduction**.
*   **Initial Step:** Reduce the cost matrix by subtracting the minimum element of each row from that row, then doing the same for columns. The sum of subtracted values is the **Lower Bound (Cost)** for the root node.
*   **Branching:** For each city $j$ from current city $i$:
    1. Set row $i$ and column $j$ to $\infty$ in the matrix.
    2. Set $M[j, \text{start}]$ to $\infty$.
    3. Reduce the new matrix and calculate the new cost: `NewCost = ParentCost + M[i,j] + ReductionSum`.
*   **Least Cost Search:** Always expand the node with the lowest bound.

### Section C: Step-by-Step Worked Example (The Application)

**Example: Solve TSP using Branch & Bound (4 Cities)**
Adjacency Matrix:
```text
   A  B  C  D
A [∞, 2, 5, 7]
B [2, ∞, 8, 3]
C [5, 8, ∞, 1]
D [7, 3, 1, ∞]
```
1.  **Reduce Matrix (Root Node):**
    *   Row mins: A:2, B:2, C:1, D:1. Total = 6.
    *   Subtract row mins.
    *   Col mins: All 0.
    *   **Cost(Root) = 6**.
2.  **Branch to City B from A (Node 2):**
    *   Set Row A = ∞, Col B = ∞. Set M[B,A] = ∞.
    *   Cost = Cost(Root) + M[A,B] + Reduction = 6 + 0 + 0 = **6**.
3.  **Branch to City C from A (Node 3):**
    *   Cost = 6 + 3 + 0 = **9**.
4.  **Least Cost Search:** Node 2 has lowest cost (6). Expand Node 2.

**Final Path:** A $\to$ B $\to$ D $\to$ C $\to$ A. **Total Cost: 11**.

### Section D: How to Write This in the Exam (The Strategy)
*   **Start With:** State that TSP is NP-Hard and we use LC-Search (Least Cost) to find the optimal tour.
*   **Body:** Show the **Row Reduction** and **Column Reduction** matrices for Node 1. Then calculate the Costs for child nodes (Node 2, 3, 4). Draw the State Space Tree clearly showing the cost at each node.
*   **Close With:** Final tour path (e.g., $1 \to 2 \to 4 \to 3 \to 1$) and the total min cost.

## Dynamic Programming Characteristics (Principle of Optimality)
**[PART A FOCUS | HIGH FREQ]**

### Section A: What is this? (The Concept)
Imagine you are finding the fastest way to drive from Delhi to Mumbai via Jaipur. 

> **The Analogy:** The Principle of Optimality says: "If the fastest route from Delhi to Mumbai passes through Jaipur, then the part of that route from Jaipur to Mumbai *must also* be the absolute fastest way to get between those two specific cities." You can't have a shortcut in a sub-section of an already perfect path.

### Section B: Exam-Ready Theory (The Rigor)
*   **Principle of Optimality:** An optimal sequence of decisions has the property that whatever the initial state and decision are, the remaining decisions must constitute an optimal decision sequence with regard to the state resulting from the first decision.
*   **Overlapping Subproblems:** The same subproblems are solved multiple times. DP solves them once and stores the result (Memoization).
*   **Optimal Substructure:** The optimal solution to a problem can be constructed from optimal solutions of its subproblems.
*   **Bottom-Up Approach:** Solutions are computed starting from the smallest subproblems and building up to the main problem.

### Section C: Step-by-Step Worked Example (The Application)

**Example: Shortest Path Property**
Consider a path $P$ from $u$ to $v$ passing through $w$: $P = (u \dots w \dots v)$.
1.  Assume $P$ is the shortest path from $u$ to $v$.
2.  Let $P_1$ be the sub-path from $u$ to $w$ and $P_2$ be the sub-path from $w$ to $v$.
3.  If there existed a shorter path $P_2'$ from $w$ to $v$ than $P_2$, then we could combine $P_1$ and $P_2'$ to get a shorter path from $u$ to $v$.
4.  This contradicts our assumption that $P$ was the shortest.
5.  Therefore, $P_2$ must be the shortest path from $w$ to $v$. This demonstrates **Optimal Substructure**.

### Section D: How to Write This in the Exam (The Strategy)
*   **Start With:** "A problem is said to satisfy the Principle of Optimality if the subsolutions of an optimal solution are themselves optimal solutions for their subproblems."
*   **Body:** Mention the two key requirements for DP: **Overlapping Subproblems** and **Optimal Substructure**.
*   **Close With:** Give a 1-sentence example, like the Shortest Path problem or Matrix Chain Multiplication.
